# Known conformance failures for ProtoBufDescriptors v1, against the Google
# protobuf conformance_test_runner from protobuf v25.9.
#
# Consumed by `conformance_test_runner --failure_list`. Each uncommented
# line is a `Required.*` test we currently fail; the runner treats them
# as expected failures so the suite reports success when only these are
# failing (and a *new* failure shows up as unexpected, which is the
# regression signal CI cares about).
#
# Categories (112 total — was 277 before the map-entry decoder fix,
# which cleared all 165 ValidDataMap.* cases):
#
#   * ~30 PrematureEof* / IllegalZeroFieldNum_*: malformed binary inputs
#     the spec wants rejected; our codec silently accepts and skips.
#     Same gap shows up as "JsonOutput" mirrors when the malformed
#     binary feeds a JSON output request.
#
#   * ~17 ValidDataScalar.*: BOOL with a varint payload >1 not accepted
#     as `true` (spec: any non-zero is true); proto2 ENUM zero-default
#     drop; UINT32 wire edge cases.
#
#   * RepeatedScalarMessageMerge / RepeatedScalarSelectsLast (~6) and
#     ValidDataOneof.MESSAGE.Merge / ValidDataRepeated tail (~10).
#     Last-wins / merge-as-singular for repeated scalars / submessages
#     when interleaved with unknowns.
#
#   * JSON-input edges (~15): whitespace inside number-as-string,
#     Float / Double overflow, Duration / Timestamp range, AnyNested,
#     ValueAcceptNull, OneofFieldDuplicate, RepeatedFieldWrongElementType,
#     EnumFieldUnknownValue.
#
# The underlying gaps:
#
#   (a) Lenient parser — accepts malformed inputs the spec rejects.
#   (b) BOOL strictness — rejects nonzero varints other than 1.
#   (c) JSON edge-case strictness — whitespace, overflow, WKT bounds.
#
# All real spec-conformance bugs; allowlisted so the harness lands green
# and any *new* failure surfaces as a regression. Tracked for follow-up
# codec passes.
#
