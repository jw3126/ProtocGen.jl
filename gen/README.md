# gen/

Self-generated Julia bindings used at runtime by `ProtocGen`,
plus the `.proto` sources they're produced from. The entire tree is
generated by our own codegen — ProtocGen self-hosts.

## Layout

- `gen/google/...` — Julia bindings, included from
  `src/ProtocGen.jl`.
- `gen/proto/google/...` — the `.proto` sources used as input.
- `gen/regen.jl` — driver script.

## Regeneration

```sh
julia gen/regen.jl
```

This runs `protoc --plugin=bin/protoc-gen-julia --julia_out=gen
--proto_path=gen/proto` against every vendored `.proto` and writes
`<name>_pb.jl` files alongside the existing aggregator
`gen/google/protobuf/protobuf.jl`. A single protoc invocation lets the
codegen Universe span all input files, which is what makes
`compiler/plugin.proto` (which imports `descriptor.proto`) work via
the cross-package alias mechanism.

The set of files regenerated:

- `descriptor.proto` (proto2) and `compiler/plugin.proto` (proto2)
  — descriptor + plugin bootstrap.
- The 11 well-known types: `any`, `api`, `duration`, `empty`,
  `field_mask`, `source_context`, `struct`, `timestamp`, `type`,
  `wrappers`. `struct` participates in a Value ↔ Struct ↔ ListValue
  cycle which codegen breaks via abstract supertypes; `api` and
  `type` reference each other across files via the Universe.

## Vendored proto sources

`gen/proto/google/protobuf/descriptor.proto` is the older
`descriptor.proto` (vendored historically with `proto3_optional`
patched in for tooling that predated the upstream addition). The
newer descriptor fields we don't carry (Editions,
`ExtensionRangeOptions.declaration`, etc.) decode as unknown fields
and are discarded — that's the proto wire contract.

`gen/proto/google/protobuf/compiler/plugin.proto` is taken verbatim
from `/usr/include/google/protobuf/compiler/plugin.proto` (libprotoc
3.20.x).

The 11 WKT `.proto` files are vendored verbatim from the upstream
protobuf repository under
`google/protobuf/{any,duration,empty,...}.proto`.

## CI invariant

Running `julia gen/regen.jl` should produce no diff against the
committed `_pb.jl` files. Any change to either codegen or a `.proto`
source must be paired with a regen.
